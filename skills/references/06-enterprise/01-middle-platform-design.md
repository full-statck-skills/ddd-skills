# Middle Platform — 中台战略与 DDD 实践

## 中台核心概念

| 概念 | 定义 |
|------|------|
| **中台** | 企业级能力复用平台，提炼各业务板块共同需求，抽象为通用业务模型 |
| **业务中台** | 通用能力和核心能力的共享服务平台 |
| **数据中台** | 全域数据汇集、融合、加工，萃取数据价值 |

## 平台 vs 中台

| 维度 | 平台 | 中台 |
|------|------|------|
| 目标 | 公共能力复用 | 企业级整体解决方案 |
| 范围 | 独立共享服务 | 前台-中台-后台全面融合 |
| 关键能力 | API 共享 | 快速响应 + 企业级复用 + 无缝联通 |
| 业务模型 | 独立系统 | 组件化产品，整体业务方案 |

## 中台与 DDD 的映射

| DDD 概念 | 中台概念 | 说明 |
|----------|----------|------|
| 核心域 | 核心中台 | 核心竞争力，最大投入 |
| 通用域 + 支撑域 | 通用中台 | 公共能力沉淀共享 |
| 限界上下文 | 微服务 | 一个 BC ≈ 一个微服务 |
| 子域 | 中台 | 中台可进一步分解为子中台 |

## 中台业务建模 5 步法

```
Step 1: 业务域分解
  └── 按业务流程或功能属性将业务域细分为多个中台
  └── 归类：核心中台 / 通用中台

Step 2: 逐中台事件风暴
  └── 找出实体、聚合和限界上下文
  └── 建立各中台的主领域模型

Step 3: 跨中台模型扫描
  └── 以主领域模型为基准，扫描其他中台
  └── 识别重复/分散的领域对象和功能

Step 4: 提炼重组
  └── 逐个主领域模型完成比对和重构
  └── 沉淀公共能力，整合分散模型

Step 5: 微服务设计
  └── 基于领域模型完成微服务设计 → 系统落地
```

## 两种建模策略

| 策略 | 做法 | 适用 |
|------|------|------|
| **自顶向下** | 顶层设计 → 逐级分解 → 建模 | 全新系统、推倒重建 |
| **自底向上** | 基于现有系统 → 对比分析 → 重组沉淀 | 遗留系统演进式重构 |

### 自底向上三步法

```
第一步: 锁定各系统所在业务域 → 分别事件风暴 → 建立各系统领域模型
第二步: 对齐业务域 → 找出同类业务的领域模型差异 → 重组重构
第三步: 中台归类 → 完成建模 → 设计微服务

核心口诀: "分域建模型，找准基准域，划定上下文，聚合重归类。"
```

## 案例：保险中台重构

### 问题

传统核心与互联网电商各建一套，导致：
- 核心能力重复（报价、投保、核保）
- 通用能力重复（用户、客户）
- 业务职能分散（缴费：传统 POS + 移动支付）

### 重构过程

```
Step 1: 分域建模
  传统核心 → 8 个领域模型（用户认证、权限、个人客户、团体客户、POS收付、报价、投保、保单管理）
  互联网电商 → 6 个领域模型（报价、投保、订单、客户、用户认证、移动收付）

Step 2: 对齐重组（以客户为例）
  互联网电商客户: 个人 + 积分（2个聚合）
  传统核心客户: 个人 + 视图 + 评级 + 归并（4个聚合）
  合并重组 → 个人 + 归并 + 视图 → 构建"个人信息"领域模型
              评级 + 积分         → 构建"评级积分"领域模型
              团体               → 直接复用"团体客户"领域模型

Step 3: 客户中台 = 个人信息 + 团体客户 + 评级积分 三个领域模型
```

## 前中后台协同

```
前台 (面向客户)
├── 营销推广、交易转化
├── 微前端动态组合、流程编排
└── 快速响应市场变化

中台 (面向运营)
├── 通用中台：认证、权限、用户、客户、订单
├── 核心中台：承保、理赔、收付
└── API 共享，服务复用

后台 (面向管理)
├── 流程审核、内控管理
├── 采购、人力、财务、OA
└── 从核心链路剥离，独立建设
```

### BFF (Backend for Frontends)

```
          前端 App
            │
        BFF 微服务 (无领域模型)
        ├── 跨中台服务编排
        ├── 渠道适配
        └── 版本与前端同步发布
       /    |    \
  中台A   中台B   中台C  (有领域模型，稳定)
```

## 中台建模常见错误

| 错误 | 纠正 |
|------|------|
| 所有领域同等对待 | 核心域重点投入，通用域可外购 |
| 简单复制原有系统模型 | 要跨系统对比分析后重组 |
| 一次建完所有中台 | 从核心域开始，逐步扩展 |
| 中台直接面向所有前端 | BFF 层做渠道适配，中台保持稳定 |
