# 15 FAQ — 常见问题

1. **分层和六边形区别？** 分层四层单向依赖；六边形通过 Port/Adapter 双向接口隔离。分层更简单。
2. **必须多模块吗？** < 5 人可以用单模块 + package 分包，但要遵守依赖规则。
3. **Domain 不能有 Spring 注解？** 是。Domain 必须是纯 POJO。
4. **值对象必须不可变？** 是。所有字段 final，构造器初始化。
5. **事务放哪层？** Application 层。Domain 方法不应有 `@Transactional`。
6. **聚合间怎么通信？** 通过 ID 引用，跨聚合操作用领域事件。
7. **Repository 返回什么？** 聚合根对象。查询用 QueryService。
8. **一个聚合多少实体？** 建议 ≤ 5 个。
9. **能和三层混用吗？** 可以，新功能四层，老功能逐步迁移。
10. **需要 CQRS 吗？** 读写模型差异大时引入 L1（模型分离）。
11. **怎么防止违规？** 用 ArchUnit 在 CI 中自动检查。
12. **适合微服务吗？** 适合。每个微服务内部用四层架构。
13. **Application Service 命名？** `{聚合}ApplicationService`，方法用业务用例名。
14. **领域服务 vs 应用服务？** 领域服务：多个实体业务协作（Domain）。应用服务：编排+事务（Application）。
15. **代码属于哪层？** 问：涉及框架？→ Infra。是编排还是业务？→ 编排→App，业务→Domain。前端交互？→ Interface。
